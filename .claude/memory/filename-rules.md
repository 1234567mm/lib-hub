# 文件名特殊字符处理规则

## 核心规则

**重要**: 文件名中的括号会导致 Docusaurus 解析失败。

## 字符替换表

| 字符 | 处理方式 | 示例 |
|------|----------|------|
| `(` | 替换为 `-` | `GPIO(使用)` → `GPIO-使用` |
| `)` | 替换为 `-` | `STM32(基础)` → `STM32-基础` |
| `（` | 替换为 `-` | `GPIO（详解）` → `GPIO-详解` |
| `）` | 替换为 `-` | `GPIO（详解）` → `GPIO-详解` |
| `#` | 替换为 `-` | `问题#1` → `问题-1` |
| `?` | 替换为 `-` | `如何做?` → `如何做-` |
| `%` | 替换为 `-` | `100%` → `100-` |
| `：` | 替换为 `-` | `标题：内容` → `标题-内容` |

## 建议的文件名格式

```
{YYYY-MM-DD}-{slug}.md
```

其中 slug 只包含：中文、字母、数字、连字符 `-`

## 应用场景

- 使用 `/new-post` 技能创建文件时，技能会自动处理
- 手动创建文件时需要注意
