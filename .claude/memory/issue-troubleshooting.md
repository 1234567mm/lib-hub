### 2026-04-19: CI 构建失败 - Sidebar docId 不存在

**问题类型**：Docusaurus 构建错误

**现象**：
```
These sidebar document ids do not exist:
- stm32/STM32知识库/STM32_RCC标准库全解析-3
- 开发工具/HermesAgent安装与模型联动
- 开发工具/Ollama安装与配置
- 开发工具/WSL2安装与基础配置
- 开发工具/知识库与项目开发工作流
```

**根因分析**：
1. 文件已创建但未 `git add` 提交到仓库（untracked 状态）
2. docId 中包含数字前缀（`01_`, `02_` 等）但 sidebars.js 引用时未包含

**修复方法**：
1. `git add docs/开发工具/` 提交新文件
2. 修改 sidebars.js，将引用改为完整 docId（包含数字前缀）
3. 或使用对象格式自定义标签：`{ type: 'doc', id: '...', label: '显示标题' }`

**预防措施**：
- 新建文档后立即 `git add` + `commit`
- 使用 `npm run start` 本地预览验证后再 push
- 修改 sidebars.js 后运行构建验证

## 问题模式库

### 模式 1：docId 与文件不匹配

**典型特征**：sidebar 引用了不存在的 docId

**预防检查**：`npm run build`

### 模式 2：untracked 文件导致 CI 失败

**典型特征**：文件本地存在但未提交，CI 环境与本地文件状态不同步

**预防检查**：
```bash
git status
```
