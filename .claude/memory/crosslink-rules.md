# 交叉链接规范

## 序列文章识别

具有以下特征的文件名视为序列文章：
- `笔记一`、`笔记二`、`笔记三`
- `Part 1`、`Part 2`、`Part 3`
- `第1章`、`第2章`、`第3章`
- `01_`、`02_`、`03_` 前缀

## 链接格式

### 新文章 frontmatter 后

```markdown
> 上一篇：[笔记一标题](/docs/stm32/入门教程/笔记一)
```

### 新文章末尾

```markdown
## 下一篇：[笔记三标题](/docs/stm32/入门教程/笔记三)
```

### 前一篇文章末尾更新

原内容：
```markdown
## 下一篇：[旧标题](/docs/stm32/入门教程/旧标题)
```

更新为：
```markdown
## 下一篇：[新标题](/docs/stm32/入门教程/新标题)
```

## 示例

创建 `笔记二` 时：

1. 在 `笔记二.md` frontmatter 后添加 `> 上一篇: [笔记一]`
2. 在 `笔记二.md` 末尾添加 `## 下一篇: [笔记三]`
3. 更新 `笔记一.md` 的 `## 下一篇` 指向 `笔记二`

## 注意事项

- 链接路径使用完整 docId
- docId 不包含 `.md` 后缀
- 路径格式：`/docs/{docId}`
