# project-showcase

**命令**：`/project-showcase`

**功能**：扫描 blog 文章，自动识别含金量高的项目，输出结构化展示数据供技术甄选页面使用。

## 用法

```
/project-showcase              # 全量扫描并更新
/project-showcase --report     # 仅输出报告
/project-showcase blog/xxx.md  # 单篇评估
```

## 评分维度

| 维度 | 条件 | 权重 |
|------|------|------|
| 标签匹配 | `tags` 含 `项目实战` | ★★★★★ |
| 标题匹配 | 标题含"项目"/"实战"/"制作" | ★★★ |
| 内容长度 | >1000 字 | ★★ |
| 代码含量 | 代码块数量 ≥3 | ★★ |
| 技术栈丰富度 | tags ≥3 个 | ★★ |

综合权重 ≥7 星认定为精选项目。

## 含金量评级

| 星级 | 标准 |
|------|------|
| ⭐⭐⭐⭐⭐ | 完整项目，原理+代码+调试，创新性强 |
| ⭐⭐⭐⭐ | 完整项目，核心代码，实用性高 |
| ⭐⭐⭐ | 项目框架+部分实现 |
| ⭐⭐ | 介绍性内容，少量代码 |
| ⭐ | 概念介绍，无实际代码 |

## 输出

- 扫描结果报告（含金量排序）
- 提示是否更新技术甄选页面的展示数据
